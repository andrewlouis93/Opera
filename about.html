About
The Optimal Performance-based Earthquake Response Analysis (OPERA) tool is a web-based application designed to enable practicing engineers design supplemental damping solutions using the Performance Spectra Methodology in accordance with the Canadian Seismic Research Network Guideline for Seismic Rehabilitation (CSRN-41) for low to medium-rise frame structures. The application is meant to provide preliminary designs using supplemental dampers. Time-history analysis should be employed to provide the ultimate performance verification for more complex systems as stipulated by the CSRN-41. To use the OPERA web application, a registration is required. If you are a registered user, the application can be accessed by logging in at the bottom of the screen. If you are a new user, click register to go to the registration screen.

Instructions

The OPERA web application generates Performance Spectra for various types of supplemental damping systems for specified ranges of period and strength of interest. For this purpose, the seismic hazard at the site of interest needs to be defined through the hazard definition module.

Hazard Definition

The design hazard needs to be defined either as a code Uniform Hazard Spectrum (UHS) or as a suite of scaled ground motion through the upload function. Standard PEER format and the two column simple format are accepted. For users interested in multi-hazard design using the Performance Spectra, it is noted that the response under a scaled hazard level defined under the hazard definition module is shown in Performance Spectra with corresponding normalized strength. While this provides a quantitative analysis of the likely global responses at other hazard levels, the change in spectral contents at different hazard levels cannot be captured.

Performance Spectra Parameters

Once the hazard is defined, the user is prompted to input the ranges of period and strength for which Performance Spectra are to be generated, as well as the base performance goals. If ground motions are uploaded in the hazard definition module, the user can choose to generate Performance Spectra using direct time-history integrations. Otherwise, by default, OPERA uses the Simplified Performance Spectra), which are generated using equivalent linearization of damped systems.

Performance Spectra Navigation

Once generated, matrix of Performance Spectra generated in OPERA can be navigated. The solution space meeting or exceeding the performance goals will be highlighted. By moving the mouse cursor over the solution space, the design target supplemental properties and normalized responses are displayed in the display pane. The display pane also shows the normalized responses under other hazard levels for multi-hazard designs. A design can be selected by double-clicking a performance point on the Performance Spectrum. This activates the Preliminary Design button on the bottom right, which takes the user to a SDOF to MDOF transformation module that assigns supplemental damping properties to each storey of the structure according to previously defined performance goals.

Preliminary Design

The preliminary design module prompts the user to define basic transformation rules in order to distribute supplemental damping to achieve the desired performance goal. The default transformation rule is equal inter-storey drift under a first mode dominant assumption. More detail can be obtained in CSRN-41.
